# Propose Doc Edits

When proposing updates to project docs, follow this pattern for each edit:
- Show the current text (before)
- Show the proposed replacement (after)
- Explain why this change is needed
- **Wait for user confirmation** before writing each change

Never write without user confirmation. Show concrete proposed edits (diff-style before/after), not vague suggestions. Keep edits surgical — update specific sections, don't rewrite entire docs.
