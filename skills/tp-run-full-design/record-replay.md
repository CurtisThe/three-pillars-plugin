# Record/Replay Hooks â€” `tp-run-full-design`

This document describes the record and replay hooks for the
`tp-run-full-design` orchestrator. The orchestrator is prose, not an
executable, so the hooks are described here as integration points; the
testable implementation lives in the `skills/_shared/trace_*.py` modules.

See also: `skills/_shared/trace_writer.py`, `skills/_shared/trace_replay.py`,
`skills/_shared/trace_index.py`.

---

## (a) Record hook site

**Flag**: `--record` on the orchestrator invocation.

The record hook fires at the **on-return clip sequence** inside
`## Return clipping` (SKILL.md). This is the OD-7 envelope-faithful capture
point â€” the orchestrator records only the already-clipped envelope, never the
raw subagent reply or its scratch reasoning.

The precise sequence, executed after each slot subagent returns:

1. Read `subagent_tokens` from the Agent return metadata (existing step 1 â€”
   SKILL.md ## Return clipping item 1).
2. Parse the reply with `parse_tier_return(reply, schema_path)` to get the
   clipped envelope (existing step 2 â€” item 2). Discard the raw reply (item 3).
3. **Record hook fires here** (new, `--record` only):
   ```python
   writer.write_slot_record(invocation_id, clipped_envelope)
   writer.emit(RETURN, invocation_id=invocation_id,
               payload={"tokens": subagent_tokens, "status": status,
                        "verdict": clipped_envelope.get("verdict")})
   writer.emit(SLOT_EXIT, invocation_id=invocation_id, payload={})
   ```
4. Continue with the existing branch-on-status decision (item 4).

The `TraceWriter` is opened once at run start (alongside `decisions.md` init)
and closed at run end via its context-manager `__exit__`. It emits `RUN_START`
on `__enter__` and `RUN_END` on `__exit__`.

**Before each slot dispatch** (also under `--record`):
```python
inv_id = invocation_id(slot_name, attempt)
writer.emit(SLOT_ENTER, invocation_id=inv_id, payload={"slot": slot_name})
# ... build prompt ...
writer.emit(DISPATCH, invocation_id=inv_id, payload={"slot": slot_name})
result = Agent(...)
```

All writes pass through `trace_filter.redact` automatically â€” `TraceWriter`
enforces this on every code path (OD-8). The `.trace/` directory is created
under the design dir:

```
three-pillars-docs/tp-designs/{slug}/.trace/{run-id}/
  meta.json           # run metadata: args, start_ts, end_ts, redactions
  trace.jsonl         # append-only event log
  slot-{inv-id}.json  # one file per slot, the clipped envelope
  blobs/              # payloads > 64 KB spilled here (content-hash named)
```

The `run-id` is a ULID-shaped 26-char Crockford base-32 string (sortable by
creation time); generated by `trace_writer.run_id()`.

Trace dirs are gitignored by default
(`three-pillars-docs/tp-designs/*/.trace/`). Use `git add -f` to deliberately
commit a canonical trace for CI replay.

## (b) Replay re-drive

**Flag**: `--replay <trace-dir>` on the orchestrator invocation.

The replay re-drive runs the slot loop structurally unchanged but substitutes
`ReplayCache.resolve` for every live `Agent(...)` call. No live model, MCP,
git, or `gh` calls are made during replay â€” it is strictly side-effect-free.

**Initialization** (once, before the slot loop):
```python
cache = trace_replay.load(Path(trace_dir))
replay_writer = TraceWriter(design_dir, args={"mode": "replay"})
```

**Each slot** (replacing the live `Agent(...)` call):
```python
inv_id = invocation_id(slot_name, attempt)
replay_writer.emit(SLOT_ENTER, invocation_id=inv_id, payload={"slot": slot_name})
clipped_envelope = cache.resolve(slot_name, attempt)   # no live call
replay_writer.write_slot_record(inv_id, clipped_envelope)
replay_writer.emit(RETURN, invocation_id=inv_id,
                   payload={"status": clipped_envelope.get("status"),
                            "verdict": clipped_envelope.get("verdict")})
replay_writer.emit(SLOT_EXIT, invocation_id=inv_id, payload={})
```

The replay `TraceWriter` writes into a **new** `.trace/<replay-run-id>/` dir â€”
the original recorded trace is never modified.

**After the loop**, compare the two trace dirs:
```python
diffs = trace_replay.diff(recorded_dir, replay_writer.run_dir)
if diffs:
    # render with trace_index; canary tripped
```

`trace_replay.diff` compares `verdict`, `status`, and `artifact_path` for
every slot in the recorded trace. An empty list means the replay trajectory
matches the recording exactly.

## (c) Cache-miss policy

**Default: strict-abort.**

When `cache.resolve(slot, attempt)` is called for a slot that is not in the
recorded trace, `ReplayCache.resolve` raises `ReplayCacheMiss`:

```
ReplayCacheMiss: replay-cache-miss design-audit#2
```

This is the safe default. A cache miss during replay almost always means the
invocation being replayed is structurally different from the recorded run (a
slot was added, an attempt count changed, or the recorded trace is stale).
Strict-abort prevents silent divergence.

**Escape hatch: `--replay-passthrough`.**

Pass `strict=False` to `resolve` (exposed via the `--replay-passthrough` CLI
flag) to receive the `MISS` sentinel instead of an exception. The orchestrator
may then fall back to a live `Agent(...)` call for that slot. This reintroduces
token spend and side effects for the missed slot; it is documented as a
deliberate escape hatch for partial-replay scenarios.

```python
result = cache.resolve(slot_name, attempt, strict=False)
if result is trace_replay.MISS:
    result = Agent(...)   # live fallback (--replay-passthrough only)
```

Use `--replay-passthrough` when:
- Replaying a run that was recorded before a new slot was added.
- Selectively re-running a single slot with live model output.

Do **not** use it as the default â€” it defeats the reproducibility guarantee
that makes the canary meaningful.

## (d) Record-to-replay canary recipe

The canary verifies that a recorded trace round-trips cleanly through the
replay machinery. It is an offline regression seed â€” it catches structural
divergence in the replay path without requiring a live orchestrator run.

**Automated canary** (test fixture):

The test `test_trace_canary.py::test_roundtrip_canary` builds a minimal
recorded trace dir using `TraceWriter`, drives a replay through
`ReplayCache.resolve` into a second `TraceWriter` dir (no live calls), and
asserts `diff(recorded_dir, replay_dir) == []` â€” the green canary state. A
deliberately mutated slot verdict asserts the tripwire fires (non-empty diff).

**Manual canary recipe** (against a real recorded run):

```bash
# 1. Record a run:
/tp-run-full-design my-design --record

# 2. Find the trace dir (most-recent ULID sorts last):
ls three-pillars-docs/tp-designs/my-design/.trace/ | sort | tail -1
# â†’ e.g. 01J9XXXXXXXXXXXXXXXXXXXXX

TRACE=three-pillars-docs/tp-designs/my-design/.trace/01J9XXXXXXXXXXXXXXXXXXXXX

# 3. Replay against the recorded trace:
/tp-run-full-design my-design --replay $TRACE

# 4. Inspect divergence (if any):
python3 "$TP_ROOT"/skills/_shared/trace_index.py $TRACE --diff <replay-trace-dir>
```

A clean canary (no diff output) confirms the replay path is faithful to the
recording. A non-empty diff identifies which slot's verdict, status, or
artifact path diverged â€” the first signal that the orchestrator's behaviour
changed between the two runs.

**Canary CI integration (optional)**:

Commit a canonical trace with `git add -f
three-pillars-docs/tp-designs/my-design/.trace/...` and add a CI step that
runs `--replay` against it. The step passes iff `diff(...) == []`. This
converts any structural change to orchestrator behaviour into a failing check.
